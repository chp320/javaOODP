프록시 패턴 사례

- 용량이 큰 이미지와 글이 같이 있는 화면을 로딩한다고 가정
- 텍스트는 크기가 작아 빠르게 로딩 가능하지만, 이미지는 크기가 커서 상대적으로 느리게 로딩 된다
- 텍스트와 이미지를 모두 로딩한 후에 화면에 노출한다면, 사용자는 긴 시간 기다려야 한다
-> 로딩이 먼저 끝난 텍스트를 우선 노출하게 하고, 이미지도 로딩이 끝나면 화면에 노출하게 한다.
-> 이를 위해 텍스트 처리용 프로세스, 이미지 처리용 프로세스를 각각 두고 구현한다.

예제 코드
- Image.java
- RealImage.java
- ProxyImage.java
- ProxyPatternMain.java
-> ProxyPatternMain 에서 RealImage 클래스에 직접 접근하지 않고, ProxyImage 에서 객체 생성 후 처리한다.
